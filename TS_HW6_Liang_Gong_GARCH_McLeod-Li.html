Here I am trying to model the volatility () of df['Return'].

The variable garch22 represents a GARCH(2,2) model fitted to the training set of log returns.

  • arch_model(train, mean="Zero", vol='Garch', p=2, q=2)
    • vol='Garch': Specifies that we are using a GARCH model (Generalized Autoregressive Conditional Heteroskedasticity).
    • mean="Zero": Assumes that the expected return is zero, meaning we only model volatility.
    • p=2: This is the ARCH(2) component, meaning past squared residuals up to lag 2 are used to predict volatility.
    • q=2: This is the GARCH(2) component, meaning past conditional variance (volatility) up to lag 2 affects future volatility.

Thus, garch22 is a GARCH(2,2) model that captures volatility clustering and persistence up to 2 lags in both past shocks (ARCH terms) and past volatility (GARCH terms).

Covariance estimator: robust

To correctly compare predicted volatility (variance) with the actual observed volatility, we should transform test data to represent actual volatility.

A common approach is to use squared returns as a proxy for realized variance.

Evaluate the models using the test set

In[11]:
def evaluate_model(model, test_data):
    predictions = model.forecast(horizon=len(test_data))
    pred = predictions.variance.values[-1, :]  # Predicted variance (volatility^2)
    
    # Compute actual realized variance (squared returns)
    actual_variance = np.square(test_data)  
    
    # Compute errors
    mse = mean_squared_error(actual_variance, pred)
    rmse = np.sqrt(mse)

    return mse, rmse

# Evaluate different models
mse_arch1, rmse_arch1 = evaluate_model(arch1, test)
mse_arch2, rmse_arch2 = evaluate_model(arch2, test)
mse_garch11, rmse_garch11 = evaluate_model(garch11, test)
mse_garch22, rmse_garch22 = evaluate_model(garch22, test)

The ARCH(1) model demonstrates the best predictive accuracy based on Mean Squared Error (MSE) and Root Mean Squared Error (RMSE):

  • MSE (2.82): Lowest among all models, indicating the best fit.
  • RMSE (1.68): Smallest error spread, confirming superior predictive performance.

Model Complexity and Fit

  • ARCH(1): A simple model with one lag of squared residuals, reducing the risk of overfitting while effectively capturing volatility.
  • ARCH(2) & GARCH models: Though more complex, they do not improve predictive accuracy, as seen in higher MSE and RMSE values.

Conclusion

For SPY ETF returns from January 2019 to April 2024, ARCH(1) provides the best balance between simplicity and predictive accuracy, effectively modeling volatility with a single lag of squared residuals.

1. Is garch22.resid Simply the Actual Returns?

No, but it's very close.

Since we set mean="Zero", the model assumes the expected return is 0, meaning the residuals are just the raw returns with no trend removed:
[ \text{Residuals} = \text{Actual Returns} - \text{Predicted Mean} ] Since the mean is 0, we get: [ \text{Residuals} = \text{Actual Returns} ]

So in this case, the residuals look like the actual returns, but they still serve an important purpose in volatility modeling.

2. Then, Whats the Point of Using GARCH(2,2)?

The key purpose of GARCH(2,2) is not to predict the returns themselves but to model and forecast volatility (conditional variance).

Here's what GARCH(2,2) does:

  • It assumes returns have periods of high and low volatility (volatility clustering).
  • It estimates how much today's volatility depends on past volatility and past shocks.
  • Instead of focusing on predicting returns, it predicts how volatile returns will be in the future.

The model gives us two key outputs:

  1. garch22.resid (Residuals):

    • These are the unexpected shocks in returns.
    • In this case, since we set mean="Zero", they look like actual returns.

  2. garch22.conditional_volatility:

    • This is the estimated volatility () of returns at each time step.
    • This is the real output of the model! It tells us when the market is in a high-volatility or low-volatility state.

1. Understanding garch22.resid (Residuals in GARCH)

  • The residuals (garch22.resid) are not exactly "shocks" in the GARCH sense but rather the difference between actual returns and the model's predicted mean.
  • Since we set mean="Zero", the predicted mean is always 0, so the residuals are just: [ \text{Residuals} = \text{Actual Returns} ]
    • This is why the residuals resemble actual returns.
    • However, GARCH doesn't just model the residualsit models how the variance of residuals changes over time.

2. Why Shouldn't Residuals Still Contain ARCH Effects?

  • Before fitting GARCH: The raw returns (or residuals in a simpler model) should show strong ARCH effects (volatility clustering).
  • After fitting GARCH: If the model is good, garch22.resid should behave like white noisemeaning no more volatility clustering should remain.

Thats why we perform the McLeod-Li test on garch22.resid.

  • If the test still finds ARCH effects, it means the GARCH model has not fully explained the volatility pattern.
  • If no significant ARCH effects remain, the model has successfully captured the volatility structure.

3. Why Doesnt This Make the Model Useless?

The true output of a GARCH model is not the residuals (resid)its the conditional volatility (conditional_volatility), which represents the estimated standard deviation at each time step.

Key takeaway:

  • garch22.resid should become white noise if the model is good.
  • garch22.conditional_volatility is the real volatility forecast we care about.
  • A good model will make the McLeod-Li test on resid non-significant, meaning no remaining ARCH effects.
  • garch22.resid should become white noise if the model is good because GARCH models volatility, not returns.
  • GARCH removes volatility clustering, meaning the residuals should no longer show patterns of changing variance.
  • While garch22.resid may still have some randomness, its variance should no longer be autocorrelatedwhich is why it should resemble white noise.

Final Answer: What Should You Expect from McLeod-Li Test on garch22.resid?

  • If garch22 is a good model No significant ARCH effects in residuals (p-value > 0.05).
  • If garch22 is not sufficient Significant ARCH effects still present (p-value < 0.05), meaning you might need a more complex model like GARCH(3,2) or an asymmetric model like EGARCH.

Summary

  1. garch22.resid should resemble raw returns but should no longer show volatility clustering.
  2. The true output of GARCH is garch22.conditional_volatility (not resid).
  3. A good model removes ARCH effects from residuals (confirmed by McLeod-Li test).
  4. If McLeod-Li test finds ARCH effects, GARCH(2,2) is not enough, and a more complex model is needed.

The values (residuals) still look similar to returns.
